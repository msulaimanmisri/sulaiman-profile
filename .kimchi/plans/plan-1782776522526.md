# Plan: Upgrade to Vite 8 + Tailwind CSS v4 (replace Bootstrap)

## Goal
Migrate `sulaiman-profile` from the legacy Vite 4 / Vue 3.2 / Bootstrap 5 (CDN) stack to the latest stable Vite 8 / Vue 3.5 / vue-router 5.1 / Tailwind CSS v4 (via `@tailwindcss/vite`) stack, with a modernized visual refresh and a `@headlessui/vue` Disclosure-based mobile menu, leaving zero Bootstrap references in the code or built output.

## Constraints
- No TypeScript conversion — keep the codebase in plain JavaScript (matches user's "upgrade to latest version of their tech stack" intent).
- Keep all current routes and views (`home`, `about`, `stack`, `project`, `personal-project`, `posts/index`, `posts/create`, `posts/edit`).
- Keep `axios` (used by `src/api/index.js` and `src/views/posts/index.vue`) — no swap to `fetch`.
- No new icon libraries; use inline SVG or text where needed.
- Do not introduce PostCSS or `autoprefixer` — Tailwind v4's Vite plugin handles CSS via Lightning CSS.
- Do not modify `public/` assets (favicon, images) or `data/`.
- `vercel.json` SPA rewrite stays as-is.
- All Bootstrap class names, `data-bs-*` attributes, and CDN `<link>`/`<script>` tags must be removed from source.

## Decision Log
- **Vite 8.1.0** (latest stable, June 2026) chosen over staying on 4.x. User explicitly asked for "latest version".
- **vue-router 5.1.0** (latest stable, May 2026) chosen per user — accept the major-version risk. Router config in this app is minimal (one file with two routes), so breaking-change risk is low. If something breaks at build time, fallback is `4.6.4` (last v4 release).
- **Tailwind CSS v4** with the first-party `@tailwindcss/vite` plugin — no PostCSS pipeline, no `autoprefixer`, no `tailwind.config.js` by default (CSS-first config via `@theme`).
- **Headless UI Vue (`@headlessui/vue`) Disclosure** for the mobile menu — user-selected. Adds one dependency; gives accessible, keyboard-friendly toggle behavior for free.
- **No icon library** — keep the bundle minimal; use inline SVG or text.
- **Keep `axios`** — already wired into `src/api/index.js` and used by `posts/index.vue`. User did not ask to migrate off it.
- **No TypeScript migration** — user said "upgrade to latest version of their tech stack", interpreted as same language, bumped versions.
- **About-page prose update** — yes, per user.
- **Preserve all `id` attributes on cards** in `personal-project.vue` — those are likely used by external links; no data was found in the repo to confirm, but preserving them is the safer default.

## Risks
- **vue-router 5.1 may introduce breaking changes** that affect this app (likelihood: low — this app's router config is small and uses stable APIs). Mitigation: if Chunk 1 reveals a router startup error, pin `vue-router` to `^4.6` instead and note in Decision Log.
- **Tailwind v4 class-name differences** — e.g. Bootstrap's `display-2` doesn't map 1:1 to a single Tailwind utility; we pick `text-6xl font-extrabold` etc. (likelihood: low — purely visual). Mitigation: visual smoke test in Chunk 6.
- **Headless UI Vue major version** — currently at v1 (stable for Vue 3). v2 may exist; we pin `^1` and let npm pick the highest 1.x.
- **Lightning CSS browser floor** — Tailwind v4 requires Safari 16.4+, Chrome 111+, Firefox 128+ (per migration guide). Mitigation: note this in the README if the project targets older browsers. The Vercel deployment does not signal older-browser requirements.
- **External links to `#bacasekejap`, `#kirasekarang`, etc.** in `personal-project.vue` — preserved verbatim in Chunk 5 to avoid breaking inbound anchors.
