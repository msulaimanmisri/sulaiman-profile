### Chunk 6 — Motion Choreography Pass (IntersectionObserver + Magnetic Buttons)
**Scope:** Wire scroll-reveal animations and magnetic button physics globally. Every primary element animates into place.
**Files Changed:**
- `src/composables/useReveal.js` (new) — Vue 3 composable wrapping `IntersectionObserver`. Returns a `ref` and a `revealed` ref. Element starts with class `motion-reveal` (defined in `main.css`), observer adds `is-revealed` on intersection. Threshold `0.15`, rootMargin `-50px`. Once-only (no un-observe, but disconnect on unmount).
- `src/composables/useStaggeredReveal.js` (new) — same as `useReveal` but applies a CSS variable `--stagger-delay` (computed as `index * 80ms`) so each child reveals 80ms after the previous one. Used in the nav modal and bento grids.
- `src/directives/magnetic.js` (new) — Vue directive. On mousemove, translates the element by `(mouseX - centerX) * 0.2` and `(mouseY - centerY) * 0.2` via `transform: translate3d`. On mouseleave, resets to `0,0` with `transition: transform 600ms cubic-bezier(0.32,0.72,0,1)`. Disabled on touch devices via `@media (hover: hover)`. Registered globally in `src/main.js`.
- `src/main.js` — register the `v-magnetic` directive globally. No new dependencies.
- All CTAs across `home.vue`, `about.vue`, `stack.vue`, `project.vue`, `personal-project.vue`, `App.vue` — apply `useReveal` on their container and `v-magnetic` on the buttons.
- All bento cards (`StatCard.vue`, the stack categories, the archive grid cells) — apply `useStaggeredReveal` on the parent grid so children fade in 80ms apart.
**Depends On:** Chunk 1 (motion-reveal class), Chunk 3-5 (the elements to animate)
**Accept When:**
- Scrolling down any page, every major block fades up + blurs in over ~1000ms with the custom cubic-bezier.
- Buttons within the bento cards and the CTAs in the nav translate slightly toward the cursor on hover (verified in desktop Chrome).
- Touch devices: magnetic effect is fully disabled (no `transform` mutation).

2. **Stat cards with index numbers** (`01`, `02`, `03`) — borrowed from editorial print design; turns abstract metrics into a typographic sequence.
3. **Stack page with three-tier mastery dots** (Chunk 4) — instead of a flat list, signals honest seniority: "daily" vs "comfortable" vs "exposure." Senior engineers don't pretend to be experts at everything.
4. **Archive grid with monospaced numbering** (Chunk 5) — turns a wall of project names into a typographic release archive (like a band's discography). Dense but readable.
5. **Filter chips by sector/category** (Chunk 5) — turns the 42-client list from "a wall" into a queryable surface. Adds interaction without clutter.
6. **Surface `/posts` writing route** (Chunk 7) — currently orphaned in the router. Even a minimal "Writing" link + latest 2-3 titles gives you thought-leadership surface area at zero ongoing cost.
7. **Magnetic button physics** (Chunk 6) — converts a static CTA into a small piece of theatre. Desktop only, but the kind of detail senior engineers notice in each other's portfolios.
8. **Philosophy principles card** in the About bento (Chunk 4, Card D) — turns the bio from CV-bullets into a stance. Three short principles signal engineering judgment, not just engineering experience.
9. **Editorial italic for emphasis words** in H1s (e.g., "& Software Architect" in Instrument Serif italic) — a single typographic detail that signals taste.
10. **Last-updated mono caption on the Stack page** — adds a subtle honesty cue ("Updated 2026-06") that signals maintenance discipline.

These are not bolted on at the end — they're structural to the chunks. When the plan executes, the design itself delivers the improvements.
